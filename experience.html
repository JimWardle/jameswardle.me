Experience

DevOps Engineer – Mercury

I joined Mercury to help improve how we deploy and manage software across a multi-tenant Azure environment. I led the move to Azure VM Scale Sets for more reliable and scalable releases, replacing hosted build agents with customised VMSS-based ones to gain full control over tooling.

One of my biggest contributions has been automating Dynamics environment updates and client offboarding processes — removing manual steps, reducing risk, and saving time. I work closely with developers in daily stand-ups, mentor junior engineers, and keep documentation sharp and useful.

Tools and areas of focus: PowerShell, Azure DevOps, YAML pipelines, Microsoft Dynamics, SQL/Cosmos DB, Datadog, and blob storage. The team runs on a two-week release cycle, and I often lead the release process end to end.

DevOps Engineer – Coventry University

Over four years at Coventry, I looked after Azure DevOps pipelines, Jenkins infrastructure, and CI/CD workflows across multiple teams. I supported API platforms like MuleSoft, administered cloud services, and handled release processes for both internal and vendor-supported systems.

Alongside the technical work, I also supported developers with Git workflows, managed SQL and PL/SQL queries, and kept legacy systems running during a slow transition to modern tooling.
